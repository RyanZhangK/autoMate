How autoMate works

autoMate is a small server running on your machine. It exposes a unified entry point that any LLM can call. When you (or an upstream model like Claude Code) hand it a natural-language request, it plans, picks the right tool, and runs it locally.

30-second quickstart
1
Configure a model

Open the tab, pick any provider, paste an API key, click "Use this". Don't have one? Each card links to the provider's key page. For free, run a local model with Ollama.

2
(Optional) Connect tools

The tab has 31 SaaS connectors. autoMate stays local — keys are Fernet-encrypted on disk and never leave your machine. OAuth supported for GitHub, Notion, Slack, Linear, 飞书, 钉钉.

3
Chat

The tab is your command line. Type plain English (or Chinese, or anything). The event stream shows every tool call so you can see exactly what's happening.

Use your real browser

The browser.* tools open a fresh empty Chromium tab via Playwright. To drive your actual browser — with your tabs, your cookies, your logins — install the autoMate Bridge extension. After that, the bx.* tools (bx.click, bx.extract, bx.navigate, etc.) work against your real session.

  1. Find the extension/ folder where you installed autoMate (or download autoMate-extension.zip from Releases).
  2. Open chrome://extensions, toggle Developer mode.
  3. Click "Load unpacked", select the folder.
  4. The toolbar badge flips to a green ON when paired.

Hook up Claude Code / Cursor / Cline / Kimi

autoMate also speaks MCP (Model Context Protocol). Add this to your upstream client's MCP config so it sees autoMate's whole tool catalog as native tools: 

{
  "mcpServers": {
    "automate": { "command": "automate", "args": ["mcp"] }
  }
}

The upstream model gets one prompt-shaped automate(prompt) tool plus every individual tool. They plan, we execute.

Privacy & safety

  • Server binds to 127.0.0.1 only — not reachable from the network.
  • API keys and OAuth tokens are encrypted with Fernet; the symmetric key lives at ~/.automate/secret.key (chmod 600).
  • The LLM call goes directly from autoMate to the provider you chose. Nothing else sees it.
  • Heads-up: shell.exec and script.run run with the autoMate process's full privileges. Treat it like any other automation tool — only run prompts you'd run yourself.
